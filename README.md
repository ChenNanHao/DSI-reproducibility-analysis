# DSI Reproducibility Analysis — repository update

This update adds the corrected analysis/statistics scripts and the final figure-generation
script that match the numbers and figures reported in the current manuscript
("Investigate ODF- and tract-based reproducibility on Diffusion Spectrum Imaging (DSI)
protocol in different field strengths and spatial resolutions"). The original scripts
already in this repository are kept — several of them are still load-bearing steps in the
pipeline — but their statistical outputs have been superseded where noted below.

## Corrected vs. original scripts

Several original scripts computed the right quantities but used a statistical procedure
that has since been corrected. The originals are kept in the repository because later
scripts still depend on intermediate files they produce (e.g. `ODF results.mat`), but the
group-comparison statistics and association models now reported in the manuscript come
from the `_v2` / newly added scripts, not from the original post-hoc/test choices:

- **`ODF_based_repro_comparisons.m` → `ODF_based_repro_comparisons_v2.m`**: the original
  used Tukey-Kramer post-hoc following a Kruskal-Wallis omnibus test (a means-based
  procedure mismatched to a rank-based omnibus test) and a hardcoded resolution comparison
  that silently dropped subjects. `_v2` replaces the dataset-comparison post-hoc with
  Dunn's test (Holm-Bonferroni corrected) and the resolution comparison with a Friedman
  omnibus + Wilcoxon signed-rank post-hoc (Holm-Bonferroni corrected), using all subjects.
  The data-aggregation portion of the original script (reading per-subject DSI Studio
  output into `ODF results.mat`) is unchanged and still required.
- **`SNR calculation 20220510.m` → `SNR_calculation_v2.m`**: the original referenced a
  `E:\` drive path that no longer exists on this system and had never been run to
  completion after the drive letter changed. `_v2` fixes the path; the underlying SNR
  computation is unchanged. Group-comparison statistics for SNR were split out into
  `SNR_PVE_wDICE_dataset_resolution_stats.m`, using the same corrected Dunn's/Friedman
  procedure as above.
- **`SNR_PVE_regression.m` / `PVE_vs_repro.m` → `SNR_only_analysis.m` / `SNR_only_analysis_wDICE.m`**:
  the manuscript's current SNR–reproducibility association (Table 1, Table 2) reports a
  linear mixed-effects model (`Reproducibility ~ SNR + (1|Subject)`) with pooled and
  resolution-stratified marginal/conditional R², computed by these two new scripts. The
  original PVE-inclusive regression scripts remain in the repository for provenance but
  are not the source of any number currently reported in the manuscript.

## Not included in this update

- Raw acquisition data, intermediate `.mat` files, and per-subject DSI Studio outputs are
  not part of this repository (data volume). These are available upon reasonable request
  from the corresponding author, per the manuscript's Data Availability statement.
- Figures 1 (acquisition/preprocessing workflow) and 2 (voxel-wise SIMODF/AngleDev maps)
  are assembled directly from acquisition/processing outputs and are not generated by a
  standalone script in this repository.
- A parallel line of analysis modeling partial volume effect (PVE) alongside SNR was
  explored during manuscript preparation and is not included here, consistent with the
  manuscript's Limitations statement that PVE was not modeled in this study.

## Software dependencies

- DSI Studio (ODF/tractography reconstruction)
- FSL (distortion, eddy-current, and motion correction)
- CONN toolbox (rigid-body co-registration)
- MATLAB Statistics and Machine Learning Toolbox (`kruskalwallis`, `fitlme`, `friedman`)
